# Инструкция по загрузке проекта на GitHub

## Шаг 1: Создание репозитория на GitHub

1. Войдите в свой аккаунт GitHub
2. Нажмите кнопку "+" в правом верхнем углу
3. Выберите "New repository"
4. Укажите имя репозитория (например, `airline-app`)
5. Выберите публичный или приватный репозиторий
6. **НЕ** добавляйте README, .gitignore или лицензию (они уже есть в проекте)
7. Нажмите "Create repository"

## Шаг 2: Инициализация Git в проекте

Откройте терминал в корневой директории проекта и выполните:

```bash
cd ~/Desktop/AirlineApp

# Инициализация Git (если еще не инициализирован)
git init

# Добавление всех файлов
git add .

# Первый коммит
git commit -m "Initial commit: Airline App with unit tests and CI/CD"

# Добавление remote репозитория (замените YOUR_USERNAME и REPO_NAME)
git remote add origin https://github.com/YOUR_USERNAME/REPO_NAME.git

# Переименование ветки в main (если нужно)
git branch -M main

# Отправка на GitHub
git push -u origin main
```

## Шаг 3: Проверка CI/CD

После загрузки проекта:

1. Перейдите на вкладку "Actions" в вашем репозитории GitHub
2. Должен запуститься workflow "CI/CD Pipeline"
3. Дождитесь завершения всех тестов и сборки
4. Проверьте, что все шаги выполнены успешно (зеленые галочки)

## Что включает CI/CD Pipeline

- **Тестирование**: Автоматический запуск unit тестов для всех 6 микросервисов
- **Сборка**: Создание Docker образов для всех сервисов
- **Проверка**: Тестирование docker-compose конфигурации

## Полезные команды Git

```bash
# Проверка статуса
git status

# Добавление изменений
git add .

# Коммит изменений
git commit -m "Описание изменений"

# Отправка на GitHub
git push

# Получение обновлений
git pull
```

## Важные файлы для GitHub

- `.github/workflows/ci.yml` - конфигурация CI/CD
- `.gitignore` - игнорируемые файлы
- `TESTING.md` - документация по тестированию
- `README.md` - основная документация

## Примечания

- Не коммитьте файл `.env` с реальными паролями и ключами
- Все тесты должны проходить перед merge в main ветку
- При создании Pull Request автоматически запустятся тесты
